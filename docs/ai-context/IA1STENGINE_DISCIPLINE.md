# IA-1stEngine Discipline

This document is the short research point for the framework discipline used by the base stack.

## What it is

- An engineering operating system for AI-assisted software development.
- A persistence layer for context, specs, decisions and execution history.
- A governance model that keeps AI outputs incremental, validated and traceable.

## What it is not

- A prompt dump.
- A big-bang refactor engine.
- A replacement for engineering judgment.

## Core rules

1. Read context before writing code.
2. Read the relevant specs before changing behavior.
3. Use routed agents and specialized skills.
4. Prefer small, validated changes.
5. Update changelog, decisions and plans when contracts change.
6. Keep runtime governance, rollback and observability visible.

## Default behavior

- Preserve architecture and boundaries.
- Avoid improvisation when a documented contract already exists.
- Keep the repository as the memory of the project.
